The Division (/) operator

When working with integers, the C division opertor (/) rounds towards zero

  • (4/2) → 2
  • 2 * (5/2) → 4
  • -5/2 → -2

The Module operator (%)

The operator produces the remainder after integer division 

/* 
  9 % 2 => 1
  8 % 2 => 0
  9 % 5 => 4
*/

Avoid using this operator with negative integers.

C identifiers

Every function, variable and structure requires an identifier. We are required to use underscore style only

Anatomy of a function definition

int my_add(int a, int b){
  return a + b;
}
  • braces indicate the beg/end of a function block
  • ensure whitespace between parameter/arguments in a function definition.

Static Type System

C uses a static type system which means that all types must be known before the program is run and the type of a given identifier cannot change.

Function terminology

We call a function by passing it arguments and we say that a function returns a value.

Functions without parameters

Use void keyword to identify a function without parameters 

int my_num(void){
  return my_add(40 ,2);
}

my_num() // 42

When actually calling that function which takes no parameters, don’t pass void.

Function Documentation

Provide a purpose statement for every function that;

  • Demonstrates the function without the including the types, then a brief description of what it does.
  • Shows an example of it being called.
  • Has require statement if necessary.
// my_divide(x,y) Returns x/y using 
//    normal integer division.
// requires: y is not 0.

int my_divide(int x, int y){
  return x / y;
}

CS 136 Style

int my_add(int a, int b){
  return a + b;
}
  • Classic placement of blocks
  • Indentation of 2,3,4 (Consistent)
  • Spaces after commas and around arithmetic operations.

When dealing with large numbers of parameters, large expressions or long purpose statements continue (indented) on the following line. 

// my_super_long_function(a,b,c,d,e,f,g) does some things...

int my_super_long_function(int a, int b, int c, int d,
                           int e,int f,int g){
  return a * b + b * c + c * d + d * e + e * f +
         f * g + g * a;
}

Entry Point (Main)

Every C program must have one and only one main function which is the entry point

The main function has no parameters and an int return type. 

int main(void){
  // ... code ...
  return 0;     // success!
}

The return value communicates to the OS the “error code” (or exit code). You don’t have to explicitly state the return value, the default is success (zero) and is returned automatically.

A successful program returns zero (no error)

Top-level expressions.

In C, top level expressions(code outside of a function) are not allowed, Code will only execute within a function

Tracing expressions

Tracing expressions can be helpful for debugging expressions and “tracing” them 

int main(void){
  trace_int(1 + 1);
  trace_int(my_add(1,2));
}

1 + 1 => 2
my_add(1,2) => 2

Function Ordering

Always place function definitions above any other functions that reference them (this would imply that main should always be at the bottom of your code)

Boolean Expressions

In C, boolean expressions do not produce “true” or “false”. The produce either;

  • zero (0) for “false”
  • one (1) for “true”

However constants “true” and “false” have been defined for us in the cs136 library.

Comparison Operators

The equality operator in C is ==

  • (3 ==3) \implies true
  • (3 ==2) \implies false

The not equal operator is !=

  • (3 !=3) \implies false
  • (3 !=2) \implies true

The other inequality operators are similar in nature.

Logical Operators

The logical operators are : ! (not), && (and), || (or)

C short-circuits and stops evaluating an expression when the value is known and a logical conclusion is reached.

All non-zero values are considered true

Operators that expect a boolean value will consider any non-zero value to be true.

Bool Type

The bool type is an integer that can only have a value of 0 or 1.

Assertions

assert(exp) consumes a boolean expression and stops the program if the expression evaluates to false (zero), continues otherwise. 

assert(my_add(1,2) == 3); // Program will continue

Function Requirements

The assert function is also very useful for verifying function requirements, and we MUST assert any feasible function requirements. 

// my_divide(x,y) Returns x/y using 
//    normal integer division.
// requires: y is not 0.

int my_divide(int x, int y){
  assert(y); // assert that y != 0
  return x / y;
}

Multiple Requirements

With multiple requirements, it is better to split them up over individual assertions. 

// my_function(x,y,z)....
// requires: x is positive
             y < z

int my_function(int x, int y, int z){
  assert(x > 0);
  assert(y < z);
}

Statements

int my_divide(int x, int y){
  assert(y);       \\ statement
  trace_int(y);    \\ statement
  return x / y;    \\ statement
  trace_int(x);    \\ unreachable statement
}

Statements are executed in sequence (one after the other).

The return statement end the function.

Conditionals

We can use if statements to control the flow of our function behaviour.

NOTE: There can be more than one return in a function, but only ONE is ever actually returned

(the function stops when the first return is executed, wherever that may be in the code block).

Ternary Conditional Operator

Enables quick execution of common boolean expressions 

(v >= 0) ? v : -v     // abs(v)
(a >= b) ? a : b     // max(a, b)
