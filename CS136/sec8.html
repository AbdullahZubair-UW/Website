Section 8 - Efficiency

Running Time

For an algorithm, we denote the running time of the algorithm by \(T(n)\) where \(n\) usually denotes the size of the input. To analyze the running time of algorithm, we subdivide into a worst and best case running time scenario. We can take a look at a simple example below, 

int sum_arr(int a[], int len){
  int sum = 0;
  int i = 0;
  while(i < len){
    sum = sum + a[i];
    i++;
  }
  return sum;
}

In the above algorithm, we would say that \(T(n) = 7n + 3\) is the running time of the algorithm. (We assume \(n\) to represent the length here)

Worst Case Running time

We are almost always exclusively concerned with the worst case running time of an algorithm, and we ignore the best case.

Big O Notation

In practice, to communicate the efficiency of an algorithm we restrict ourselves to referring the order of an algorithm, which is similar to the concept of the order of a polynomial (But not always). For example if \(T(n) = a_{m}n^m + a_{m - 1}n^{m - 1} + \ldots + a_0\), then we would say that the order of the running time is \(O(n^m)\). Here are a few examples,

  • \(O(1) = 4\)
  • \(O(n^2) = n^2 + n + 4\)
  • \(O(n) = n + 5\)
  • \(O(n\log n) = n + n\log n\)
  • \(O(n^3) = \frac{n(n + 1)(2n + 1)}{6} = \sum_{i = 0}^n i^2\)
  • \(O(2^n) = n + 2^n\)

Big O Arithmetic

Addition

For addition, the result is simply the largest order

  • \(O(\log n) + O(n) = O(n)\)
  • \(O(1) + O(1) = O(1)\)

Multiplication

For multiplication, the result is the product of the two orders (And you can give a combinatorial argument as to why)

  • \(O(\log n) \times O(n) = O(n\log n)\)
  • \(O(1) \times O(n) = O(n)\)

Iterative Analysis

When analyzing the run time of an iterative program, we use a summation, 

int simpl(int a[], int len){
  int res = 0;
  for(int i = 0 ; i < len; i++){
    if(a[i] % 2 == 0){
      res++;
    }
  }
  return res;
}

In the above code, we analyses the run by;

  • Determine the bounds of the summation
  • Determine the running time per iteration
  • Write the summation.

In our example, \[T(n) = \sum_{i = 1}^n O(1) = O(1) + O(2) + \ldots + O(n) = O(n)\]

Common Summations

  • \[\sum_{i = 1}^{\log n} O(1) = O(\log n)\]
  • \[\sum_{i = 1}^{n} O(1) = O(n)\]
  • \[\sum_{i = 1}^{n} O(n) = O(n^2)\]
  • \[\sum_{i = 1}^{\log n} O(i) = O(n^2)\]
  • \[\sum_{i = 1}^{n} O(i^2) = O(n^3)\]

Recursive Analysis

When analyzing the run time for recursive functions, refer to the look-up table to choose the correct corresponding match,

Run time of Sorting algorithms

Selection Sort

void selection_sort(int a[], int len){
  int pos = 0;
  for(int i = 0 ; i < len; i++){
    pos = i;
    for(int j = i + 1; j < len; j++){
      if(a[j] < pos){
	pos = j;
      }
    }
    swap(&a[i], &a[pos]);
  }
}

We conclude that the run time will be, \[T(n) = \sum_{i = 1}^n \sum_{j = 1}^n O(1) = O(n^2)\]

Insertion Sort

void insertion_sort(int a[], int len){
  for(int i = 1; i < len; i++){
    for(int j = i; j > 0 && a[j - 1]> a[j]; j--){
      swap(&a[j], &a[j - 1]);
    }
  }
}

We conclude that the run time will be, \[T(n) = \sum_{i = 1}^n \sum_{j = 1}^i O(1) = O(n^2)\] Note that the best case running time is when the array is already sorted, which is \(O(n)\). The worst case will be \(O(n^2)\).

Quick Sort

In the best case, the pivot is dead centre, in which case \[T(n) = O(n) + 2T(n / 2) = O(n\log n)\]

In the worst case, the pivot is smallest (or largest) element and hence we split into a set of size \((n - 1)\) and another that is empty. Hence, \[T(n) = O(n) + T(n - 1) = O(n^2)\]

Formal definition of Big O

Definition: \(O(g(n))\) is the set of all functions whose polynomial order is less than or equal to \(g(n)\).
