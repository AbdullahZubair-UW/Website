Section 3 - An Intro to C

Functional Programming

Functional Programming paradigm: Functions are pure, as in,

  • Functions only return values
  • Return values only depend on argument values.
  • Only constants are used.

Imperative Programming paradigm: for(int i = 0 ;i < n; i++) Functions are impure, as in,

  • Variables are used.
  • Mutation.
  • Sequences of statements are executed in a block.

Imperative Programming

Programs are mostly sequences of statements to be executed. The significant defence between functional programming and imperative programming is that imperative programming produces side effects

Text I/O

To display output in C, we use the printf.

printf

  • In order to properly format code output to appear on multiple lines, use the newline character "\n" 

         printf("Hello \nC can be\ntricky")

     /*

     Hello
     C can be
     tricky

       */

  • To output an integer expression, use the “%d” format specifier. 

         printf("2 + 2 is : %d\n", 2 + 2) // 2 + 2 is : 4

    For multiple integer expressions, follow each expression in order with a comma and include a corresponding “%d”

  • To output an actual percent sign (%), use (%%). 

         printf("You got %d%% on your test", 100) // You got 100% on your test
  • To output an actual backslash (\), use (\\).
  • To output an actual quote (“), use (\”).
  • Note: Printf does return a value, it returns the number of characters in the string its passed.

Functions with Side Effects

Programming side effects are when the state of something changes. Some side effects include,

  • Output to the console

We must document any side effects that our functions may produce. printf itself is a function that produces a side effect (changes the display or output) and hence any function that calls printf produces side effects.

Testing I/O

Recall that any assertion based testing will only check if the program returns the correct value, it does not check wether it produced the correct or desired output. For this we can resort to Seashell I/O testing or we can implement some customized technique to test from the terminal.

Void Functions

Functions that return no values are typed as void. The return statement is optional.

Statements

There are only three types of C statements (in the plural sense).

Compound statements (blocks {}).

A sequence of statements to be executed.

Expression statements

Statements for generating side effects.

Control Flow statement

Control the order in which other statements are executed. (Eg. return, if, else)

Examples of Side Effects

The primary types of side effects we will be analyzing are;

  • Output
  • Mutation (Modifying variables)
  • Input

Variables

Variables store values, to define them we use the following syntax;

  • The type
  • The identifier

  • The initial value 

        int my_var = 7 // Definition

    Variable definitions are not statements. Note that the (=) in intilization is not the same as in mutation.

Mutation

When the value of a variable is changed, it is known as mutation. We achieve mutation using the assignment operator (=).

Assignment operator (=)

Used to preform mutation. It also produces the right hand side value, in other words we can preform a multi-assignment as follows; 

    x = y = z = 0; // (x = (y = (z = 0)))

In general, avoid having more than one side effect per expression statement.

Constant

A constant is a variable that is immutable. 

    const int my_const = 7 // Definition

Global and Local Varibles

Global Varibles

  • Varibles that are defined outside of functions

Local Varibles

  • Variables that are defined inside of functions
  • Have Block Scope, in other words their scope and accessibility extends from definition until the end of the block they are defined within.
    int my_global_var = 7;

    void func(void){
      int my local_var = 8;
      // ...
    }

Mutating Global/Local Variables

  • A function that mutates global variables has a mutation side effect and therefore must be documented.

  • A function that mutates a local variable defined within scope is not a mutation side effect, and hence remains pure. For example the following is a pure function; 

            int add1(int n){
          int k = 0;
          k += 1;
          return n + k;
        }
  • Note: Parameters are treated like local variables and hence if a function mutates them within scope then it remains pure.

Global dependency

A function that depends or uses the value of global mutable variables is considered impure although it may not have any side effects.

Testing Harness

For functions that only return values, you may build a so called testing harness that prints the return values of the function based on a given set of input, 

    // sqr(n) Returns the square of the integer n.

    int sqr_n(int n){
      return n * n
    }

    // test_sqr() An I/O testing harness for sqr, continously reads input and prints
    //      sqr(n) for each input.
    // effects: reads input
    //          produces output

    void test_sqr(void){
      int n = read_int();
      if (n != READ_INT_FAIL){
        printf("%d\n", sqr(n));
        test_sqr(n);
      }
    }

    int main(){
      test_sqr();
    }
